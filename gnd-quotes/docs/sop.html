GND Optus Civil Quoting — SOP

Standard Operating Procedure & Human Discipline Guide
Version 1.0 · 26 Mar 2026 · Damian sign-off required for amendment

1. Purpose

This document is the single human-readable reference for how GND quotes Optus fibre civil works using the GND Quote Cowork system. It covers the entire lifecycle: receiving a drawing, generating a priced quote, separating internal governance from the commercial submission, maintaining quote history, and the quarterly audit cycle.

The AI system (Claude Cowork) executes phases automatically based on instruction files in the Instructions/ folder. This SOP governs the human steps that surround, gate, and verify the AI's outputs.

5. Two-File Output Separation Protocol

This protocol is mandatory for every bid. It is not optional and does not change with job size or complexity.

Every priced quote run produces exactly two derivative files in Output/:

FileNamingContainsAudience
Master Internal Control [ORDER]_Master_Internal_Control_[DATE].xlsx Every annotation, margin view, rate-card flag, variation register, V-B findings, knowledge capture. Header banner: "INTERNAL — DO NOT SHARE" Damian, Karan only
Commercial Submission [ORDER]_Submission_Clean_[DATE].xlsx Golden-template-compliant. Zero flags, zero unresolved items, zero margin data. Minimal "Job Context" sheet with 3 neutral risk bullets. Optus / PE / PM

Sanitisation Rules (Submission_Clean only)

  • Remove all colour-coded flags (🔴, ⚠, ❓), "MISSING QTY/RATE", "HARD STOP", "RATE CARD ERROR"
  • Remove provisional $0 values for unconfirmed items
  • Replace detailed quote notes with: (a) design-version statement if required, (b) GST-exclusion note, (c) one neutral sentence re supplied drawings
  • Delete all internal governance sheets (Margin View, Variation Risk Register, etc.)
  • Insert minimal "Job Context" sheet — 3 bullets: design version, third-party assets (no owners/$ exposure), council/permit notes
  • Re-verify all subtotal/total formulas chain correctly after deletions

Separation Verification Checklist

  • Submission_Clean contains zero buy rates
  • Submission_Clean contains zero flags or annotations
  • Submission_Clean total matches Internal Control sell-side total
  • No master files in Inputs/, Reference/, or Golden_Templates/ were modified
  • Both files reside exclusively in Output/
Human gate: Before final submission, a human reviewer must confirm the Submission_Clean file is "drama-free" while the Internal Control file remains "forensically complete."

6. RC4 Quote History — Data Integrity

RC4 tracks what Optus actually received. Logging the internal full-risk number corrupts every downstream market signal.

The Rule

The "GND Quoted ($)" value in GND_RC4_Quote_History.xlsx must always be the TOTAL QUOTED PRICE from the _Submission_Clean file — never the Internal Control total.

Update Workflow (Karan — after every submission)

  1. Open the _Submission_Clean file from Output/
  2. Copy the final TOTAL QUOTED PRICE
  3. Open Reference/GND_RC4_Quote_History.xlsx
  4. Paste the total into Column G (GND Quoted) for the relevant row
  5. Enter the submission date in Column B
  6. Add a one-line note in Key Notes explaining any deliberate deviation from the internal total (e.g. "Clean submission used conservative bore estimates")
  7. Save. Market Signal Summary auto-updates.

Sequencing (AI-generated RC4 row)

In a /full run, Phase 7 generates a provisional RC4 row before Phase 8 (Output Assembly) runs. Phase 8 emits a confirmation comparing the Submission_Clean total against the Phase 7 provisional. If they differ, Phase 8 provides the corrected RC4 row. Karan uses whichever is final.

7. Rate Card Maintenance

  1. After Damian returns a /validate review, Karan updates any flagged rates in the rate cards (RC1-SELL Activities, RC2 Materials), entering the date in the version header
  2. Damian signs off. Karan re-runs /quote to generate the updated quote
  3. Any rate change in either RC1-SELL or RC1-COST is reviewed by Damian before the next quote run
  4. Gap register items (❓ MISSING) are added to the rate card before submission. Recurring items are promoted to the Master BoM
Rate card drift: RC1-SELL and RC1-COST must never diverge. Any update to one requires verification of the other. The margin calculation is only reliable when both cards are synchronised.

8. Knowledge Capture & /learn Protocol

  1. Phase 7 auto-generates a knowledge capture entry at the end of every /full run — outcome marked PENDING
  2. After Optus responds, Karan triggers /learn [ORDER] and provides:
    • Win or Loss
    • Winning bid amount (if known)
    • How the job played out vs the drawing
  3. The entry is formatted for paste into the GND Market Intelligence Doc, with tags for suburb, council, PE, carrier, and complexity tier
  4. An RC4 row is also generated for paste into Quote History

9. Quarterly Audit

Every quarter, Damian and Karan conduct a review covering:

9a. RC4 Integrity Audit

  • Every "GND Quoted ($)" value in RC4 matches the corresponding archived _Submission_Clean file in Output/
  • Any mismatch triggers an immediate RC4 correction — log the Submission_Clean total, add a correction note
  • Win/Loss and Winning Bid columns are populated for all closed jobs
  • Market Signal Summary reflects accurate data

9b. Rate Card Audit

  • RC1-SELL and RC1-COST are aligned — no unreviewed divergence
  • All gap register items from the quarter have been resolved or excluded with rationale
  • Market Overrides from the quarter are reviewed: were they justified? Should any become permanent rate changes?
  • Subcontractor rates (buy rates) are still current — no structural rate inversions

9c. Folder & File Hygiene

  • No files in Inputs/, Reference/, or Golden_Templates/ were modified outside the documented amendment protocol
  • Every job in Output/ has both _Master_Internal_Control and _Submission_Clean files
  • Golden template is still current — update version if Optus format requirements changed

9d. Knowledge & Intelligence Review

  • All closed jobs have a completed Knowledge Capture entry (Phase 7)
  • Market Intelligence Doc entries from the quarter are reviewed — any PE behaviour changes, council pattern shifts, or carrier infrastructure updates
  • Any new failure modes or anti-patterns identified → proposed as Constitution amendments (Damian proposes, Karan drafts, Damian signs off)

10. Five Constitutional Principles (Reference)

These govern every AI output. They are reasoning principles, not rules. The AI executes them silently — this section is for human understanding.

  1. The Margin Firewall — Buy rates (RC1-COST) never appear in any external output. The spread between sell and buy is the business. Folders are immutable.
  2. Provisional Sums Are Not Excuses — Every provisional sum needs a specific trigger, resolution action, and gate wave. "TBC" is not a resolution.
  3. Hard Stops Are Hard — GND does not accept commercial responsibility for elements it cannot control at fixed price. Hard Stops are never softened in language.
  4. Irreversible Evidence Windows — Depth of cover, cable serials, bore GPS, duct allocation. Miss them and GND does not get paid. They are payment conditions, not compliance niceties.
  5. The Two-Audience Rule — Karan/Damian get tables with numbers. Craig gets imperative checklists in plain English, on a phone, standing up.

11. Amendment Protocol

  1. Damian proposes the change
  2. Karan drafts the amendment in the relevant instruction file
  3. Damian signs off
  4. Version number increments. Previous version archived, not deleted

For Constitution-level changes (Hard Stop definitions, margin firewall, new principles): Damian sign-off is mandatory. For Router or Ingestion tuning (output formats, anomaly patterns, productivity rates): Karan and the Claude expert can implement, with Damian review if it affects Hard Stop classification.
